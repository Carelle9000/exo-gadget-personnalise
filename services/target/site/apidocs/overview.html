Description générale du projet : Gadget Personnalisé pour Dashboard eXo Platform

Ce projet de fin de stage vise à développer un gadget personnalisé pour l'interface dashboard de eXo Platform. Le gadget affiche :

  • La date et l'heure actuelle.
  • La météo en fonction de la position géographique de l'utilisateur (obtenue via géolocalisation et appel API backend).
  • Les derniers documents traités par l'utilisateur (récupérés via APIs internes eXo).
  • Une citation motivante par jour (via API externe proxyée en backend).
  • Un message de bienvenue personnalisé basé sur l'utilisateur connecté.
L'objectif est d'améliorer l'expérience utilisateur et d'accroître la productivité en fournissant des informations pertinentes et motivantes directement sur le dashboard.

Structure du projet : Multi-modulaire Maven avec un module service (JAR) pour le backend (appels API externes/internes) et un module webapp (WAR) pour le frontend (gadget OpenSocial avec HTML, CSS, Tailwind CSS). Tous les appels API sont gérés en backend pour respecter les normes de sécurité et d'intégration eXo.

1. Vue d'ensemble

Ce gadget est basé sur la spécification OpenSocial supportée par eXo Platform. Il utilise un backend Java pour gérer les appels API (météo via OpenWeatherMap, citations via quotes.rest, documents via eXo REST APIs). Le frontend est un gadget OpenSocial XML avec contenu HTML/JS, stylisé avec Tailwind CSS pour une interface responsive et moderne.

2. Architecture et Structure Multi-Modulaire

Le projet est structuré en Maven multi-modules :

  • Parent POM : Définit les dépendances communes (eXo kernel, OpenSocial, etc.) et les modules enfants.
  • Service Module (JAR) : Contient les services backend Java, REST endpoints (utilisant JAX-RS) pour proxyer les appels API externes et internes.
  • Webapp Module (WAR) : Contient le fichier gadget XML, ressources statiques (HTML, JS, CSS compilé Tailwind), et configuration pour déploiement dans eXo.

3. Installation et Configuration

Prérequis : JDK 21, Maven 4.0.0, eXo Platform installée via Docker.

  • Clonez le repository : git clone "https://github.com/Carelle9000/exo-gadget-personnalise".
  • Construisez : mvn clean install.
  • Déployez le JAR service dans eXo.
  • Déployez le WAR dans le serveur eXo.
  • Ajoutez le gadget au dashboard via admin eXo.
  • Configurez les clés API dans service/src/main/resources/config.properties.

4. Utilisation

Ajoutez le gadget à votre dashboard eXo. Le gadget chargera automatiquement les données via appels JS aux endpoints REST backend. La géolocalisation demande la permission utilisateur via navigator.geolocation.

5. Référence de l'API Backend

Endpoints REST dans le module service :

  • GET /rest/gadget/weather?lat={lat}&lon={lon} : Récupère la météo.
  • GET /rest/gadget/quote : Récupère une citation.
  • GET /rest/gadget/documents?userId={userId} : Récupère les documents.

6. Implémentation Frontend

Le gadget est défini dans webapp/src/main/webapp/gadget.xml avec HTML, JS, et Tailwind CSS pour une UI moderne.
